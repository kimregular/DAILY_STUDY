# when 조건문

## 인자를 받는 when 조건문

코드의 실행을 제어하는 방법을 제어 흐름이라고 하는데, 자주 사용하는 if, else 조건문이 바로 대표적인 제어 흐름이다. 코틀린도 같은 if, else를 지원하지만 또다른 제어문이 있다. 바로 when 조건문이다. when은 자바의 swtich와 유사하지만 훨씬 더 많은 기능을 제공한다. 다음은 자바의 switch 예제다.

```java
// 자바
public String getTypeOfDayWithSwitchStatement(int dayOfWeek) {
	String typeOfDay;
	switch(dayOfWeek) {
		case 1:
			typeOfDay = "한 주의 시작";
			break;
		case 2:
		case 3:
		case 4:
			typeOfDay = "주중";
			break;
		case 5:
			typeOfDay = "한 주의 끝";
			break;
		case 6:
		case 7:
			typeOfDay = "주말";
			break;
		default:
			throw new IllegalArgumentException("잘못된 날: " + dayOfWeek);
	}
	return typeOfDay;
}
```

이제 동일한 역할을 하는 함수를 코틀린으로 구현해보자.

```kotlin
// 코틀린
fun getTypeOfDayWithSwitchStatement(dayOfWeek: Int): String {
	val typeOfDay: String = when (dayOfWeek) {
		1 -> "한 주의 시작"
		2, 3, 4 -> "주중"
		5 -> "한 주의 끝"
		6, 7 -> "주말"
		else -> throw IllegalArgumentException("잘못된 날: $dayOfWeek");
	}
	return typeOfDay
}
```

### when으로 enum 클래스 다루기

각 색상(enum class)에 해당하는 한글을 가져오는 함수를 만들어보자.

```kotlin
fun getKoreanColor(color: Color): String = 
	when(color) {
		RED -> "빨강"
		ORANGE -> "오렌지"
		YELLOW -> "노랑"
		GREEN -> "초록"
	}
```

역시나 여러 enum class를 조건으로 갖는 when 함수를 만들 수도 있다.

```kotlin
fun getKoreanColor(color: Color) = 
	when(color) {
		RED, ORANGE -> "빨강 또는 오렌지"
		YELLOW -> "노랑"
		GREEN, BLUE -> "초록 또는 파랑"
	}
```

- enum의 모든 구성요소를 나열하지 않거나 `else`가 없으면 컴파일 오류가 발생한다.

## 인자를 받지 않는 when 조건문

```kotlin
fun mix(c1: Color, c2: Color) = 
	when {
		c1 == RED && c2 == YELLOW -> "오렌지"
		c1 == YELLOW && c2 == BLUE -> "초록"
		else -> throw RuntimeException()
	}
```

- if else if else 문장과 동일한 구조이다.
- 조건식이 길지 않고 유사도가 높은 경우 간혹 사용한다.

## 스마트캐스트

`when`으로 타입 검사와 타입 캐스트를 동시에 할 수 있다!

```kotlin
fun printObject(obj: Any) {
	when(obj) {
		is String -> println(obj.lowercase()) 
		// String 변환되어 lowercase() 메서드 사용가능!
		is Duration -> println(obj.nano)
		// Duration 변환되어 nano 프로퍼티 접근가능!
		is LocalDateTime -> println(obj.month)
		// LocalDateTime 변환되어 month 프로퍼티 접근가능!
		else -> println("Unknown type")
	}
}
```
