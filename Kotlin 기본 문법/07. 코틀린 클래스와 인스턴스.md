# 코틀린 클래스와 인스턴스
자바에서는 (롬복을 사용하지 않으면) 아래와 같이 클래스에 getter/setter를 선언하여 사용해야한다.

```java
public class Person {

	private String name;
	private int age;
	private boolean isMarried;
	
	public String getName() {
		return name;
	}
	
	public void setName(String name) {
		this.name = name;
	} 
	
	public int getAge() {
		return age;
	}
	
	public void setAge(int age) {
		this.age = age;
	}
	
	public boolean isMarried() {
		return isMarried;
	}
	
	public void setMarried(boolean married) {
		this.isMarried = married;
	}
}
```

코틀린에서는 위와 동일한 코드를 단 1줄로 구현할 수 있다!

```kotlin
class Person(val name: String, var age: Int, val isMarried: Boolean)
```

- 선언부 프로퍼티 모두를 갖는 주 생성자는 자동으로 생성된다.
- 클래스 이름 옆에 나열한 것들이 모두 프로퍼티(field, getter, setter)로 동작한다.
- getter/setter를 커스텀하고자 하는 경우에만 별도로 선언한다.

사용코드를 살펴보자.

```kotlin
fun main(args: Arrays<String>) {
	val person = Person("James", 32, true)
	println(person.name)
	println(person.isMarried)
}

// James
// true
```


## 커스텀 접근자
```kotlin
class Rectangle(val height: Int, val width: Int) {

	val isSquare: Boolean
		get() {
			return height == width
		}
		// get() = height == width
}
```

- 함수(`fun isSquare(): Boolean`)로 정의할 수도 있으나, 클래스의 프로퍼티를 나타내는 값이므로 커스텀 getter를 통한 프로퍼티 값을 드러내는 것이 더 자연스럽다.
