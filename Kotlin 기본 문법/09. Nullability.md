# Nullability

- 자바에서는 Null 값을 잘못 사용하면 NullPointerException(NPE) 발생
- NPE 방지를 위해 메서드 최상단은 항상 Null 여부 체크가 필수

## Null 될 수 있는 타입 vs. Null 될 수 없는 타입

```kotlin
val nullable: String? // null 될 수 있는 타입
val notNull: String // null 될 수 없는 타입
```

- 물음표를 사용하여 nullable 타입임을 나타낸다.
- 해당 타입의 경우 컴파일 시점부터 null check 진행
	- not null type variable = nullable type variable : 컴파일 오류 발생
	- nullable type variable = not null type variable : 가능
- 함수 파라미터, 응답타입 등이나 클래스의 생성자 혹은 프로퍼티 정의에도 사용 가능하다.

### nullable 클래스 타입의 메서드 호출 경우

```kotlin
val nullablePerson: Person? = ...

// 1. if 검사
if (nullablePerson != null) {
	// if문 조건식에서 null이 아님을 체크했다면
	// 본문에서는 값에 바로 접근 가능하다
	println(nullablePerson.age)
}

// 2. ?. 호출
println(nullablePerson?.age)
// 값이 있는 경우 호출, 없는 경우 null

// 3. !!. 호출
println(nullablePerson!!.age)
// 값이 있는 경우 호출, 없는 경우 NPE
// 가급적이면 지양

// 4. ?: 사용
println(nullablePerson?.age ?: 0)
// 값이 있는 경우 호출, 없는 경우 0
```
